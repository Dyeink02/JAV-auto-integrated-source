## 0.21 重点更新说明

### 1. 新增磁力内容校验（广告过滤）

本版本新增可选开关：`磁力内容校验（广告过滤）`。

开启后，程序会在候选磁力之间做进一步筛选：

- 尝试读取磁力内部文件列表
- 判断最大文件是否为视频主文件
- 检查是否混入 `.url`、`.apk`、`.html`、压缩包、安装包等明显广告 / 杂文件
- 命中广告包特征时，自动跳过当前候选磁力
- 自动切换到下一条更合适的候选磁力，尽量减少人工二次补筛

### 2. 保守兜底，避免误伤正常结果

考虑到部分磁力元数据可能因为网络、Tracker 或超时问题而暂时无法读取，0.21 版本没有采用“读不到就直接丢弃”的激进策略，而是：

- 优先保留明确校验通过的磁力
- 如果全部候选都无法确认，则回退到首条“未验证”候选
- 这样可以在降低广告包概率的同时，尽量避免把正常结果误判为无结果

### 3. 与原有过滤词能力联动

0.21 版本的磁力处理链路改为：

1. 先执行原有的“磁力过滤词”规则
2. 再执行新增的“磁力内容校验（广告过滤）”
3. 最后根据设置返回单条最优磁力或全部保留磁力

这样可以同时满足：

- 你手动过滤指定尾缀 / 关键词
- 软件自动过滤广告包 / 杂文件包

### 4. 对客户体验的实际提升

这一版的核心目标不是增加花哨功能，而是减少你抓取后的重复劳动。

适合解决的场景：

- 同一番号多个磁力里，最大那条其实是广告包
- 磁力里混入跳转网址、说明页、安装包
- 结果表面抓到了 100 条，但其中不少仍要手工再筛一轮

0.21 的目标就是把这一步尽量前移到程序里自动完成。

## 使用建议

- 若你更关心速度，可按需关闭该功能
- 若你更关心结果纯净度，建议开启该功能
- 在代理稳定、网络较好的环境下，磁力内容校验成功率会更高
- 如个别磁力一直读不到元数据，程序会记录警告日志，便于后续排查
