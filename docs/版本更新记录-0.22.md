## 0.22 重点更新说明

### 1. 爬虫 + 视频整理真正一体化

在原有 JAV 爬虫架构不变的前提下，新增“视频整理”工作区，并保持同一套 UI 风格。  
用户可以在同一个软件中完成：

1. 爬取磁力（或图片）
2. 导出磁力文本
3. 下载后整理目录
4. 输出重命名与遗漏报告

### 2. “广告处理方式”可视化增强

整理配置中新增显式可见的“广告处理方式”选择卡片：

- `移入待删除（推荐）`
- `直接删除广告文件`

并在任务执行日志与整理结果中同步显示“直接删除数量”，减少客户误解。

### 3. 全链路对账：遗漏番号与补抓磁力

整理阶段会对照爬虫番号名录：

- 识别“应有但未下载到本地视频”的遗漏番号
- 输出遗漏番号与对应磁力补抓报告
- 方便客户快速补下，不用人工逐条核查

### 4. 广告样本学习链路增强

新增“按番号自动学习”能力：

- 客户输入番号列表（逗号或换行）
- 软件在目标目录中自动匹配视频并抓帧学习
- 支持导入广告样本与正常样本
- 学习摘要实时刷新（样本数、阈值、更新时间）

### 5. 运行可视化与日志增强

整理中新增进度总览与阶段提示：

- 扫描进度：`已扫描 X/Y`
- 待整理进度：`待整理总数 N，已执行 M`
- 广告处理进度：`待删除/已删除总数 N，已执行 M`

让客户能实时看到软件“正在做什么”。

### 6. 目录清理策略优化

整理完成后会自动清理空目录，默认仅保留 `待整理` 作为核心输出目录；其余目录为空时自动删除，避免目录结构越积越乱。

## 0.22 同版补丁（2026-04-25）

在不提升语义版本号的前提下，补充以下关键修复：

1. 全中文修复
- 学习摘要、风险证据字段与状态提示中的英文残留全部改为中文。

2. 开头广告分流修复
- 命中开头广告风险的视频统一进入 `含开头广告`，不再混入 `待删除`。
- 新增“含开头广告”相关报告口径，便于人工复核与回灌学习。

3. 根目录收口修复
- 整理完成后，根目录残留番号文件夹改为强制删除，并增加二次清理兜底。
- 目标状态：
  - `移入待删除` 模式：顶层保留 `待整理 / 含开头广告 / 待删除`
  - `直接删除` 模式：顶层保留 `待整理 / 含开头广告`

4. 完成态统计修复
- 任务完成文案新增 `含开头广告` 计数，提升客户复盘可见性。
